# Модуль «Напоминания» — важные даты клиентов

## Как это работает

### 1. Сбор дат из заказов

При добавлении заказа (Продажа сайт) с указанием:
- **Повод**: День рождения / Юбилей / Свадьба
- **Дата доставки**: когда букет доставляется (≈ дата праздника)
- **Клиент**: телефон или email

→ Дата доставки **автоматически сохраняется** в профиль клиента.

**Пример:** Заказ на доставку 15 апреля, повод «День рождения» → в профиле клиента сохраняется ДР 15 апреля.

### 2. Где хранятся даты

- Таблица `customer_important_dates` (миграция `migrations/add_customer_important_dates.sql`)
- Типы: `birthday`, `anniversary`, `wedding`, `other`
- День рождения также дублируется в `customers.birthday` для совместимости

### 3. Где отображаются

#### Вкладка «Напоминания»
- Список предстоящих праздников на 7/14/30/60/90 дней
- Фильтр по типу: Все, ДР, Юбилей, Свадьба
- Блок «Сегодня празднуют» — отдельно выделен
- Клик по строке → открывается карточка клиента
- Контакты (телефон, email) для звонков и SMS

#### Карточка клиента (вкладка «Клиенты»)
- Блок «Важные даты»: ДР, юбилей, свадьба
- Добавление дат вручную
- Удаление дат (кроме ДР из старой базы)

### 4. Напоминания каждый год

Система считает даты **ежегодно** по дню и месяцу.  
Например, ДР 15 апреля будет отображаться 15 апреля каждого года.

### 5. Миграция

Выполните SQL из `migrations/add_customer_important_dates.sql` в Supabase:

```sql
-- Создаёт таблицу customer_important_dates
-- Переносит существующие ДР из customers.birthday
```

Доступ к «Напоминаниям» — по праву доступа «Клиенты».
